# 고객 요청 취소 절차 (Customer-Initiated Cancellation)

고객이 주문 접수(Created) 혹은 배송 준비 중(Processing) 단계에서 취소를 요청하는 경우의 세부 처리 절차를 정의합니다.

---

## 주문 상태별 처리 절차

### 1. Created 상태 (즉시 취소)
-   **조건**: 주문이 아직 결제 완료 상태(`Created`)로 물류센터 지시가 내려가기 전인 경우.
-   **처리 프로세스**:
    1. 고객이 마이페이지에서 '주문 취소' 버튼을 클릭합니다.
    2. OMS는 즉시 시스템 내에서 자동 취소 및 환불을 승인 처리합니다.
    3. 결제 대행사(PG사)의 결제 취소 API를 호출하여 자동 환불 절차를 진행합니다.
-   **결과**: 주문 상태가 `Cancelled`로 즉시 변경됩니다.

### 2. Processing 상태 (승인 취소 검토)
-   **조건**: 물류센터로 이미 피킹/패킹 지시가 전달(`Processing`)된 경우.
-   **처리 프로세스**:
    1. 고객이 마이페이지에서 '주문 취소'를 요청하면 즉시 취소되지 않고 '취소 대기' 상태로 들어갑니다.
    2. OMS는 물류 창고 시스템(WMS)에 해당 주문의 **출고 중단 가능 여부**를 질의합니다.
    3. **출고 중단 성공 시**: 
        - WMS는 작업을 멈추고 보류 처리합니다.
        - OMS는 취소 승인을 내리고 주문 상태를 `Cancelled`로 변경한 뒤 환불을 진행합니다.
    4. **출고 중단 실패 시 (상차 완료 등 이미 발송된 경우)**:
        - WMS는 출고 중단 불가를 OMS에 회신합니다.
        - OMS는 고객 취소 요청을 기각하고, 고객에게 이미 배송이 시작되어 취소가 불가하며 상품 수령 후 **'반품 플로우'로 전환하여 진행**해야 함을 안내합니다.
